# PageIndex RAG 方案

## 概述
PageIndex RAG 是一种基于文档层级结构的树形索引结合 LLM 推理式检索的方案。与传统向量 RAG 不同，它无需向量数据库，保留原始文档结构，并支持跨页多跳推理。

## 核心要点

### 核心原理
- 基于文档层级结构的树形索引
- 使用 LLM 进行推理式检索
- 无需向量数据库

### 特点
- ✅ 保留原始文档结构
- ✅ 支持跨页多跳推理
- ✅ 可追溯的检索路径
- ✅ 高精度
- ✅ 部署轻量

### 性能数据
- FinanceBench 准确率：**98.7%**
- 传统向量 RAG 准确率：87.5%
- 性能提升：约 11.2%

### 优势与劣势
**优势：**
- 可追溯的检索路径
- 高精度
- 部署轻量
- 无需向量数据库

**劣势：**
- 只适合单文档场景
- 响应速度较慢
- 多文档扩展性差

### 应用场景与结合方案
| 项目规模 | 推荐方案 | 说明 |
|---------|---------|------|
| 小型项目 | 不使用 PageIndex | 直接使用传统 RAG |
| 中型项目 | PageIndex 作为精排器 | 粗排 + 精排 |
| 大型项目 | PageIndex + 向量 + BM25 | 三路召回 + RRF 融合 + Cross-Encoder 精排 |
