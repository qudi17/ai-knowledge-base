# 知识内化框架 - Learn-Verify-Apply Loop

**创建日期**：2026-02-28  
**提出者**：Eddy + Jarvis  
**版本**：v1.0

---

## 🎯 核心理念

> **学习不是输入，而是转化。知识只有被应用，才真正属于你。**

### 三大支柱

```
📥 输入 (Learn)     →  系统性吸收
   ↓
🧠 内化 (Verify)    →  主动验证掌握
   ↓
🎯 输出 (Apply)     →  现实世界应用
   ↓
📊 反馈 (Review)    →  持续改进
```

---

## 📋 四阶段流程

### 阶段 1：📥 输入 (Learn) - 系统性吸收

**目标**：高效吸收新知识，建立初步理解

#### 1.1 学习前准备

**检查清单**：
- [ ] 明确学习目标（为什么要学这个？）
- [ ] 预估学习时间（需要多少小时？）
- [ ] 选择学习材料（论文/文档/视频/课程）
- [ ] 创建学习空间（Obsidian 笔记/Notion 页面）

**模板**：
```markdown
# 学习目标卡片

**主题**：[填写]
**日期**：2026-XX-XX

## 为什么学这个？
- 解决什么问题？
- 对当前项目有什么帮助？
- 长期价值是什么？

## 预期产出
- [ ] 理解核心概念
- [ ] 能够向他人解释
- [ ] 完成一个实践项目

## 时间预算
- 总时长：X 小时
- 分配：阅读 Xh + 笔记 Xh + 实践 Xh
```

---

#### 1.2 主动阅读法

**SQ3R 方法**：
1. **Survey（浏览）**：快速浏览全文，了解结构
2. **Question（提问）**：将标题转化为问题
3. **Read（阅读）**：带着问题深入阅读
4. **Recite（复述）**：用自己的话总结
5. **Review（复习）**：回顾关键点

**笔记模板**：
```markdown
# [主题] 学习笔记

## 核心问题
- 这篇论文/文档要解决什么问题？
- 现有方法的局限性是什么？
- 作者提出的方案是什么？

## 关键概念
| 概念 | 定义 | 我的理解 |
|------|------|---------|
| ... | ... | ... |

## 核心方法/技术
1. [方法 1]：...
2. [方法 2]：...

## 关键公式/代码
[粘贴重要内容]

## 我的疑问
- [ ] 疑问 1
- [ ] 疑问 2
```

#### 1.3 知识结构化

**工具**：概念图 / 思维导图

**模板**：
```
核心概念
```

### 阶段 2：🧠 内化 (Verify) - 主动验证掌握

**目标**：确保真正理解，而非"以为自己懂了"

#### 2.1 费曼技巧验证法

**步骤**：
1. **选择概念**：选择一个刚学的概念
2. **假装教学**：想象向一个完全不懂的人解释
3. **发现 gaps**：卡壳的地方就是理解盲区
4. **返回学习**：重新学习盲区部分
5. **简化语言**：用类比和简单例子解释

**验证模板**：
```markdown
# 费曼验证卡片

**概念**：[填写]

## 我能用一句话解释吗？
[尝试用一句话概括核心]

## 我能用类比解释吗？
[找生活中的类比]

## 我能回答这些问题吗？
- 它解决了什么问题？
- 为什么这个方法有效？
- 有什么局限性？
- 和 [相关概念] 有什么区别？

## 我的理解盲区
- [ ] 盲区 1：...
- [ ] 盲区 2：...

## 行动计划
- [ ] 重新阅读 [章节]
- [ ] 查找补充资料
- [ ] 实践验证
```

---

## 💡 进阶技巧

### 1. 学习伙伴机制

**方法**：找一个学习伙伴，互相验证

**流程**：
```
1. 各自学习同一主题
2. 互相费曼验证（轮流解释）
3. 互相提问挑战
4. 分享应用场景
```

**Jarvis 可以扮演学习伙伴**：
- 提问验证你的理解
- 挑战你的假设
- 提供不同视角

---

### 2. 学习组合拳

**组合多种学习方式**：
```
阅读 (30%) + 实践 (50%) + 教学 (20%) = 最佳效果
```

**为什么**：
- 阅读：建立初步理解
- 实践：深化理解和技能
- 教学：检验和巩固

---
