# AI 技术领域分类体系

> 全面的 AI 技术分类，便于研究和学习

---

## 📊 第一维度：按目标用户/应用场景分类

### To B（企业级）
- **客服与支持**：智能客服、工单处理、知识库问答
- **办公自动化**：文档生成、邮件处理、会议纪要
- **数据分析**：BI 报表、数据洞察、预测分析
- **营销与销售**：内容生成、线索挖掘、个性化推荐
- **研发提效**：代码生成、测试用例、技术文档
- **企业搜索**：内部知识检索、语义搜索
- **合规与风控**：文档审查、风险检测、合规检查

### To C（消费级）
- **个人助手**：智能问答、日程管理、生活建议
- **学习辅导**：作业辅导、语言学习、知识问答
- **创意内容**：写作辅助、图片生成、视频编辑
- **娱乐互动**：角色扮演、游戏 NPC、聊天机器人
- **生活服务**：旅行规划、食谱推荐、购物助手
- **情感陪伴**：心理咨询、陪伴聊天、情感分析

### To Dev（开发者工具）
- **代码助手**：代码补全、重构建议、代码审查
- **API 生成**：文档生成、Mock 数据、测试用例
- **调试辅助**：错误分析、性能优化、安全检测
- **DevOps**：脚本生成、监控分析、故障诊断
- **技术文档**：API 文档、架构文档、教程生成

## 🏗️ 第二维度：按技术架构分类

### RAG（检索增强生成）
- **文档解析**：PDF、Office、网页、图片 OCR
- **向量化**：Embedding 模型、分块策略、混合检索
- **向量数据库**：Chroma、Pinecone、Weaviate、Qdrant、Milvus
- **检索策略**：相似度检索、重排序、混合检索
- **知识图谱**：图谱构建、关系抽取、图检索

### Agent（智能体）
- **单 Agent**：任务规划、工具调用、自我反思
- **多 Agent**：协作模式、角色分工、通信协议
- **Agent 编排**：ReAct、Plan-and-Solve、BabyAGI、AutoGPT
- **记忆管理**：短期记忆、长期记忆、向量存储
- **工具生态**：浏览器、代码执行、文件操作、API 调用

### Fine-tuning（微调）
- **全量微调**：全参数训练、资源需求高
- **PEFT**：LoRA、QLoRA、Adapter、Prefix Tuning
- **指令微调**：Instruction Tuning、RLHF、DPO
- **领域适配**：专业术语、行业知识、特定风格

### Prompt Engineering（提示工程）
- **基础技巧**：Zero-shot、Few-shot、Chain-of-Thought
- **结构化提示**：角色设定、任务描述、输出格式
- **提示模板**：Prompt Templates、动态构建、版本管理
- **提示优化**：APE（Automatic Prompt Engineering）、DSPy

### 混合架构
- **RAG + Fine-tuning**：检索增强 + 领域微调
- **Agent + Knowledge Base**：智能体 + 知识库
- **多模型协作**：大模型 + 小模型、任务分工
- **人机协作**：AI 辅助 + 人工审核

## 🎯 第三维度：按核心能力分类

### 语言能力
- **文本生成**：写作、翻译、摘要、改写
- **文本理解**：分类、抽取、情感分析、NER
- **对话系统**：上下文理解、多轮对话、意图识别
- **代码能力**：生成、分析、调试、重构

### 多模态能力
- **图像理解**：OCR、物体检测、图像分类
- **图像生成**：文生图、图生图、图像编辑
- **语音处理**：ASR、TTS、语音识别、语音合成
- **视频理解**：视频分类、动作识别、视频摘要
- **跨模态**：图生文、图文检索、多模态推理

### 推理与规划
- **逻辑推理**：数学推理、因果推理、常识推理
- **任务规划**：步骤拆解、任务分解、目标达成
- **复杂决策**：多目标优化、风险评估、策略选择
- **反思与迭代**：自我评估、错误修正、持续改进

### 记忆与上下文
- **短时记忆**：对话历史、上下文窗口、Token 限制
- **长时记忆**：向量存储、数据库、知识图谱
- **外部知识**：RAG、API 调用、搜索引擎
- **记忆检索**：语义检索、相关性计算、重排序

### 工具与集成
- **API 调用**：函数调用、参数解析、结果处理
- **代码执行**：沙箱环境、安全执行、结果返回
- **文件操作**：读写、搜索、格式转换
- **浏览器操作**：网页抓取、表单填写、自动化操作

## 🔧 第四维度：按基础设施分类

### 部署方式
- **云端部署**：API 服务、托管平台、SaaS
- **边缘部署**：本地运行、离线模式、隐私保护
- **混合部署**：云端 + 本地、负载均衡、弹性扩展

### 模型服务
- **模型 API**：OpenAI、Anthropic、智谱、百度文心、阿里通义
- **开源模型**：Llama、Mistral、Qwen、GLM
- **本地推理**：Ollama、vLLM、Text Generation Inference
- **量化部署**：4bit、8bit、INT8、FP16

### 开发框架
- **LangChain**：应用框架、组件化、生态丰富
- **LlamaIndex**：数据框架、RAG 专用、索引优化
- **Semantic Kernel**：微软出品、技能抽象、插件化
- **AutoGen**：多智能体框架、微软出品
- **BMad**：敏捷开发框架、工作流驱动

### 监控与可观测性
- **性能监控**：延迟、吞吐量、Token 消耗
- **质量监控**：准确性、一致性、幻觉检测
- **成本监控**：API 费用、资源使用、优化建议
- **日志追踪**：请求链路、错误分析、审计日志

## 📚 第五维度：按数据与知识管理分类

### 文档处理
- **文档解析**：MarkItDown、LangChain Document Loaders、Unstructured
- **格式转换**：PDF → Markdown、Office → JSON、OCR
- **分块策略**：固定大小、语义分块、递归分块
- **元数据提取**：标题、作者、日期、标签

### 向量化
- **Embedding 模型**：OpenAI text-embedding-3、BGE、E5、M3E
- **向量维度**：768、1024、3072、1536
- **批量处理**：批量编码、并行处理、缓存策略

### 向量数据库
- **开源方案**：Chroma、Weaviate、Qdrant、Milvus、Pgvector
- **云服务**：Pinecone、MongoDB Atlas Vector Search、Elasticsearch
- **混合检索**：向量检索 + 全文检索、重排序、融合策略

### 知识管理
- **知识图谱**：Neo4j、Neo4j Vector、知识抽取、关系构建
- **数据清洗**：去重、标准化、质量评估
- **版本控制**：文档版本、向量版本、模型版本

## 🤖 第六维度：按 Agent 能力分类

### 任务编排
- **单任务**：简单问答、文档生成、数据抽取

### 协作模式

- **多人模式**：多 Agent 协作、角色分工
- **人机协作**：AI + 人类、审核机制、反馈循环

### 记忆系统
- **短期记忆**：对话历史、当前上下文
- **长期记忆**：向量存储、结构化存储

## 🔄 使用指南

### 如何使用这个分类体系

1. **研究学习时**：
   - 根据兴趣选择一个维度深入
   - 跨维度对比不同技术方案
   - 建立知识图谱，理解技术关联

2. **项目选型时**：
   - 按应用场景定位目标类别
   - 对比同类技术方案的优缺点
   - 评估技术成熟度和生态完整性

3. **技术规划时**：
   - 按发展阶段规划技术路线
   - 平衡技术创新与稳定可靠
   - 考虑长期维护和扩展性

### 分类示例

**RAG 知识库系统**：
- 目标用户：To B（企业搜索）
- 技术架构：RAG
- 核心能力：语言能力 + 记忆与上下文
- 基础设施：向量数据库 + 文档处理
- 应用场景：知识管理

**AI 编码助手**：
- 目标用户：To Dev（开发者工具）

- 核心能力：代码能力 + 工具与集成
- 基础设施：本地推理 + IDE 集成
- 应用场景：编程与开发
